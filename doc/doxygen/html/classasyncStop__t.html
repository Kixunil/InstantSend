Detailed Description

Interface for asynchronously stopping server or peer This is optionl feature of plugin but good quality plugin should have it. It's used when user decides to unload plugin, cancel transfer or stop application. This prevents delays.

Member Function Documentation

virtual bool asyncStop_t::stop ( ) [pure virtual]

Stops waiting for connection or data If plugin is executing function, which waits for something (connection or data), calling of this function (from other thread) should cause it to return. If plugin isn't executing such function, calling of stop() should cause next call to such function return immediately, to prevent race conditions. If such function was interrupted by stop(), it should return NULL (meaning no data/connection received)

Returns:
false if stop failed (for sure), true if stop succeeded or it's unknown.
